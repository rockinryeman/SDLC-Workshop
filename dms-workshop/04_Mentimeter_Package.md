# Mentimeter Package — DMS Workshop

Build these 5 interactions in Mentimeter ([mentimeter.com/app/home](https://www.mentimeter.com/app/home)),
in order. Display the **join code / QR on every slide** so people can join anytime.

> **Setup tip:** Create one Mentimeter presentation with these as sequential slides, OR embed
> them into your main deck using Mentimeter's PowerPoint plugin so you never switch windows.

---

## Poll #1 — Opening (Multiple Choice)
**Question:** Where does your organization spend the most engineering effort?
- Requirements
- Architecture
- Development
- Testing
- Compliance / Documentation

*Use to:* warm up the room and set up the "handoffs are expensive" point.
*Place at:* ~5 min (during "Why development is slow").

## Poll #2 — What slows us down (Word Cloud)
**Question:** What slows product delivery the most today?
*(open text — let answers like "handoffs", "approvals", "requirements churn" surface naturally)*

*Use to:* generate the room's own language for the problem, then react live.
*Place at:* ~15 min (during "AI and team transformation").

## Poll #3 — Midpoint (Multiple Choice)
**Question:** Which SDLC activity benefited most from AI so far?
- Requirements
- Architecture
- Testing
- Implementation
- Deployment

*Use to:* break up the 35-min live exercise and keep energy up.
*Place at:* ~45 min (mid live-exercise, after Prompt 4 or 5).

## Poll #4 — Where to pilot (Ranking)
**Question:** Rank where you would pilot AI first.
- Systems Engineering
- Software Development
- Test Engineering
- Program Management
- Operations

*Use to:* move the room from "interesting" to "where would we actually start?"
*Place at:* ~75 min (during "Future-state team design").

## Poll #5 — Closing (Multiple Choice)
**Question:** What is the biggest opportunity?
- Smaller teams
- Faster delivery
- Better quality
- Lower cost
- More innovation

*Use to:* end on forward energy and capture the room's priority.
*Place at:* ~88 min (closing).

## Optional DMS-specific polls (swap in if you want more scenario flavor)
- **Trust poll (Scales):** *"How comfortable are you with a car judging whether YOU are too tired
  to drive?"* (Not at all → Completely) — great right before the live exercise to surface the
  ethics/trust angle.
- **False-alarm poll (Multiple Choice):** *"What's worse — a DMS that occasionally nags you for no
  reason, or one that occasionally misses real drowsiness?"* — sparks the precision-vs-safety debate.
